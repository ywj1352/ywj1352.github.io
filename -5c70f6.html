ch1 漫人生

0x00

作者极度讨厌 OOP，坚信 OOP 是一切屎山的根源，包括现在c++逼仄的定位和永不精通的刻板印象在很大程度上也是因为持续复杂的继承关系。语言本来就应该是简单的，程序员才是应该对世界和问题进行抽象并解决的个体。磨刀不误砍柴工，如同 JAVA 和 C++ 的语言，除非是具体到某个领域的专属工具，不然一定会在不断重构中浪费社会的生产效率。

Python 是一个很好的工具，但是想要用 Python 进行工程实现，最好要同sharp的，有责任心的人共事，代码的优雅应当是一种自然的追求，事实上 Python 是可以做到这一点的。当同事sharp但没有责任心时，我会倾向于选择 Golang 作为开发语言。但同事有责任心但不sharp时，个人认为 Rust 会是一个折衷的选择。

写代码应当是好玩的，所有单纯为了世俗的成功而写出来的，后续便不再维护的工程是一个程序员的耻辱。

路阻且长，共勉。
